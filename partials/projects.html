Roomies

This application is used to facilitate interactions between housemates. Roomies is built using the AngularJS framework. This application's database implements a NoSQL database through Firebase. Users are able to organize shopping trips, bill payments, chores, as well as chat with one another.
 

Technologies Used

  • HTML
  • CSS
  • Vanila JavaScript
  • AngularJS 1.x
  • Grunt
  • Firebase
  • Github and Git

Problems Encountered

While desigining the app, I originally did not realize the complex database structure that was required. In fact, I originally had very little experience using databases at all. I quickly realized that a flat database structure was essential to the performance of the application, because Firebase does not allow for relationships between database entities. That being said, I began to attach a unique primary key to each "household" then assign that key to most items in the database, this ensured that items for each house was easily found through querrying the database and searching for that specific foreign key. This repeated foreign key did make things difficult as users not only have a "house ID" assigned to them, they also have a unique "user ID". I found that sometimes I would querry the database for the improper key, as well as apply the wrong foreign key to a new instance of an object, pushing these objects into a strange database limbo, this was quickly fixed.

I found that there are some limitations when using firebase due to the none relational design of the database. However, the easy setup of the project made firebase valuable to the overall project. In the future I look forward to using other nonSQL databases.
